# 2603.28239v2.pdf - A Switch-Centric In-Network Architecture for Accelerating LLM Inference in Shared-Memory Network

## 中文标题：一种用于在共享内存网络中加速大语言模型推理的以交换机为中心的网络内架构

### 中文摘要
针对大语言模型（LLM）推理中的通信瓶颈，本文提出了一种名为 SCIN 的以交换机为中心的网络内计算架构。
现有的加速器中心化架构（如 NVIDIA 的 NVLink Sharp, NVLS）存在两大局限：1）依赖 GPU 的 Load 指令触发规
约，导致规约后的数据必须先传回发起 GPU 才能广播，引入了冗余通信；2）受限于内存语义指令，无法支持如
网络内量化（INQ）等复杂算子，导致带宽浪费。SCIN 通过在交换机内引入加速器（ISA）并结合协同设计的通
信织物，实现了低延迟和高带宽的 All-Reduce。SCIN 能够直接发起内存语义操作，消除了冗余数据移动；同时
集成了量化模块，将 All-Reduce 精度降至 8 位，几乎使带宽翻倍。实验结果显示，SCIN 在小消息和大消息下
分别实现了高达 8.7 倍和 3.8 倍的 All-Reduce 加速，使 LLaMA-2 模型的首字生成时间（TTFT）缩短了 1.74
倍，每个 Token 生成时间（TPOT）缩短了 1.34 倍。

### 各章节主旨内容
- **Chapter 1: Introduction** - 介绍 LLM 推理（特别是张量并行 TP）中 All-Reduce 的通信瓶颈，以及现有
  方案（如 NVLS）的局限性，引出 SCIN 的动机。
- **Chapter 2: Background and Motivation** - 详细分析 TP 推理中的消息大小分布、通信瓶颈，以及现有以
  加速器为中心的网络内计算架构在冗余传输和算子支持方面的局限。
- **Chapter 3: Design and Implementation** - 详细阐述 SCIN 的核心设计，包括交换机内加速器（ISA）、
  网络协议扩展、同步机制以及网络内量化（INQ）的实现。
- **Chapter 4: Evaluation** - 介绍实验方法论，通过硬件校准的模拟器和 FPGA 原型验证 SCIN 在带宽、延
  迟以及端到端 LLM 推理性能上的提升。
- **Chapter 5: Related Work** - 回顾现有的网络内计算技术和针对 HPC 或 AI 加速器的互联技术。
- **Chapter 6: Conclusion** - 总结 SCIN 的设计目标、实现成果及其对 LLM 推理效率的提升。

### 中文结论
本文提出的 SCIN 是首个针对 AI 加速器共享内存网络设计的以交换机为中心的网络内计算架构。通过消除冗余
的数据传输路径和引入低开销的网络内量化技术，SCIN 显著降低了 All-Reduce 的延迟并提升了有效带宽。在
多 FPGA 原型和大规模模拟实验中，SCIN 展示了卓越的性能提升，能够有效缓解分布式 LLM 推理中的通信墙问
题，为下一代高性能 AI 系统提供了重要的架构设计参考。

# 章节详细解读

## 第一章：引言 (Introduction)

### 1. 设计思想 (Design Philosophy)
- **痛点分析**：LLM 的规模增长使得分布式推理成为必然，而张量并行（TP）是常用手段。然而，TP 依赖大
  量的 All-Reduce 操作，这些操作处于执行的关键路径上，导致 GPU 必须停顿等待通信完成。研究表明，在
  高性能 NVLink 互联的节点中，All-Reduce 仍可能占到端到端延迟的 60%。
- **核心动机**：现有的网络内规约方案（如 NVLS）是“加速器中心化”的，由 GPU 触发数据拉取，这导致了
  不必要的回传操作。作者主张转向“交换机中心化”范式，让交换机能够主动管理通信流程。
- **设计目标**：实现同时具备低延迟（针对 decode 阶段的小消息）和高带宽（针对 prefill 阶段的大消
  息）的 All-Reduce 优化。

### 2. 关键技术 (Key Technologies)
- **交换机内加速器 (ISA)**：一个位于交换机内部的处理单元，能够理解并执行复杂的规约和量化逻辑。
- **内存语义操作 (Memory-Semantic Operations)**：使交换机能够直接访问加速器的统一全局地址空间。

## 第二章：背景与动机 (Background and Motivation)

### 1. 设计思想 (Design Philosophy)
- **痛点分析**：TP 推理包含 Prefill（预填充）和 Decode（解码）两个阶段。Prefill 产生大消息 All-Reduce，
  受带宽限制；Decode 产生多个小消息 All-Reduce，受延迟限制。现有的 NVLS 架构由 GPU 驱动，规约结果
  必须先回到 GPU 再广播，增加了网络跳数。
- **核心动机**：量化技术在计算和存储中广泛应用（如 FP8/INT8），但在 TP 通信中通常仍使用高精度（FP16/
  BF16），造成巨大的带宽浪费。

### 2. 关键技术 (Key Technologies)
- **张量并行 (TP)**：将模型权重分散在多个 GPU 上，每层需要两次 All-Reduce。
- **NVLS 的局限性**：
    1. **冗余传输**：数据规约后必须先传回发起 GPU（Store 指令），无法直接从交换机广播给所有 GPU。
    2. **算子支持受限**：无法集成复杂的算子，如在规约过程中实时进行量化压缩。

## 第三章：设计与实施 (Design and Implementation)

### 1. 设计思想 (Design Philosophy)
- **交换机中心化范式**：将控制逻辑从 GPU 转移到交换机内的 ISA。ISA 主动发出数据拉取请求（Pull），
  处理后再直接推送（Push）给所有参与者。
- **协同设计**：为了保持与现有系统的兼容性，需要对通信织物（Fabric）进行微小的协议修改。

### 2. 关键技术 (Key Technologies)
- **ISA 微架构**：包含波控制器（Wave Controller）和按加速器分布的波表（Wave Tables）。
    - **波表管理**：用于缓冲在途数据，确保按序规约。
    - **量化/反量化单元 (Quant/Dequant Unit)**：支持网络内量化 (INQ)。
- **网络协议扩展**：在数据包头仅增加 1 位“网络内计算 (INC)”标志位，用于区分 ISA 发起的事务与加速器发
  起的事务。
- **同步机制**：采用基于共享内存的轻量级同步，通过原子递增同步计数器来确认数据就绪，规约完成后 ISA
  直接更新各参与者的同步标志。

### 3. 实现细节 (Implementation Details)
- **波管理 (Wave Regulation)**：由于缓冲区有限，大消息被划分为多个“波”。ISA 允许 $k$ 个波同时在
  途，以掩盖同步延迟。
- **INQ 实现**：采用分块动态量化（Block-wise Quantization），每 64 个元素为一个块并对应一个缩放
  因子（Scale Factor）。

### 4. 权衡与对比 (Trade-offs & Comparison)
- **延迟对比**：相比 NVLS 需要两次网络跳转（规约回传+广播），SCIN 缩短为一次。
- **开销考量**：ISA 增加了交换机的逻辑面积，但在 8 节点系统中，通过减少带宽浪费和降低延迟带来的
  性能增益远超其硬件成本。

## 第四章：实验评估 (Evaluation)

### 1. 设计思想 (Design Philosophy)
- **评估目标**：验证 SCIN 在现实硬件上的可行性（FPGA 原型）以及在大规模系统中的扩展性（校准模拟器）。
- **基准方案**：与软件环形 All-Reduce (Software Ring All-Reduce) 以及 NVLink NVLS 进行对比。

### 2. 关键技术 (Key Technologies)
- **硬件原型**：使用 5 个 FPGA（4 个模拟加速器，1 个模拟交换机）构建。
- **模拟器校准**：基于 BookSim2 构建，并根据 FPGA 原型的数据进行校准（误差 < 6%）。

### 3. 实现细节 (Implementation Details)
- **模型覆盖**：评估了 LLaMA-2-7B/13B/70B。
- **配置**：TP=8，带宽 900 GB/s，链路延迟 250 ns。

### 4. 权衡与对比 (Trade-offs & Comparison)
- **微观指标**：小消息 All-Reduce 加速达 8.7 倍；大消息（启用 INQ）加速达 3.8 倍。
- **端到端指标**：LLaMA-2 模型的 TTFT 提升高达 1.74x，TPOT 提升 1.34x。
- **精度分析**：8 位 INQ 对模型 perplexity（困惑度）的影响极小（降幅约 0.0025 PPL），优于传统的环
  形量化。
