# RAPID-Graph：使用内存处理技术在图上进行动态规划的递归式全对最短路径算法

## 论文整体骨架

**论文文件名和英文标题**
*   **文件名**: `2601.19907v1.pdf`
*   **英文标题**: RAPID-Graph: Recursive All-Pairs Shortest Paths Using Processing-in-Memory for Dynamic Programming on Graphs

**中文标题**
*   RAPID-Graph：使用内存处理技术在图上进行动态规划的递归式全对最短路径算法

**中文摘要**
全对最短路径（APSP）仍然是大规模图分析的主要瓶颈，因为其立方复杂度的海量数据移动会压垮传统内存层次结构的带宽。
在本文中，我们提出了RAPID-Graph来应对这一挑战。这是一个通过算法、架构和器件级优化相结合的协同设计的内存处理（PIM）系统。
在算法层面，我们引入了一个递归感知的划分器，通过将图分解为顶点区块来减少数据依赖，从而实现精确的APSP计算，使得Floyd-Warshall和Min-Plus核心算法都能在数字PIM阵列内完全就地执行。
在架构和器件层面，我们设计了一个2.5D PIM堆栈，它在一个统一的先进封装中集成了两个相变存储器计算裸片、一个逻辑裸片和高带宽暂存存储器。
一个外部非易失性存储堆栈用于持久化存储大规模APSP的计算结果。
该设计实现了区块级和单元级的并行处理，以维持高吞吐量。在245万节点的OGBN-Products数据集上，RAPID-Graph的速度比当前最先进的GPU集群快5.8倍，能效高1186倍，
同时在速度和效率上分别超过了先前的PIM加速器8.3倍和104倍。与NVIDIA H100 GPU相比，它还实现了高达42.8倍的加速和392倍的能源节省。

**各章节主旨内容**
*   **第一章：引言 (Introduction)**
    *   阐述了全对最短路径（APSP）问题在计算上的挑战性，特别是由于其高复杂度和数据移动导致的瓶颈。
    *   指出现有GPU和PIM解决方案的局限性，并引出本文提出的RAPID-Graph作为一个软硬件协同设计的解决方案。

*   **第二章：背景 (Background)**
    *   介绍了图的表示方法、经典的Floyd-Warshall（FW）算法、分块APSP计算思想以及相变存储器（PCM）技术的基本特性。

*   **第三章：RAPID-Graph软硬件协同设计 (RAPID-Graph Software-Hardware Co-design)**
    *   详细阐述了核心的递归分块APSP算法，以及专为该算法设计的异构2.5D PIM硬件架构。
    *   描述了硬件的具体实现，包括专门用于FW和Min-Plus（MP）操作的计算裸片，以及算法在硬件上的映射和调度策略。

*   **第四章：实验结果与分析 (Experimental Results and Analysis)**
    *   展示了RAPID-Graph与CPU、GPU以及其他顶尖APSP解决方案在性能和能效上的对比结果。
    *   分析了该架构在不同图规模、节点度和拓扑结构下的可扩展性。

*   **第五章：结论 (Conclusion)**
    *   总结了RAPID-Graph作为首个为精确APSP设计的协同PIM系统所取得的成就，强调了其在性能、能效和可扩展性上的巨大优势。

**中文结论**
我们提出了RAPID-Graph，这是首个为精确APSP计算而进行软硬件协同设计的PIM系统。
通过一个递归划分器，它将子图映射到PCM阵列中，实现了完全并行的FW和MP核心算法的就地执行。
其2.5D堆栈通过一个2 Tb/s的UCIe内联器连接了两个PCM裸片、16GB 8-Hi HBM3和16TB FeNAND，在2kW的功率预算内维持了高封装带宽。
在真实和合成图上的测试表明，RAPID-Graph相比Intel i7-11700K CPU实现了高达1061倍的加速和7208倍的能效提升；
相比NVIDIA H100 GPU，则分别实现了42.8倍的加速和392倍的能效提升。在245万节点的OGBN-Products图上，它的运行时间比顶尖的GPU集群快5.8倍，能效高1186倍，
同时相比之前的PIM方案，速度和能效分别提升了8.3倍和104倍。该系统在4倍的度数变化范围内保持了平稳的性能，能够线性扩展至245万个节点，
并能从针对集群化和真实世界拓扑的局部感知分区中受益。RAPID-Graph以极低的成本、功耗和运行时间，提供了集群规模的APSP性能。

## 指定章节详解

**第一章：引言 (Introduction)**
本章首先明确了APSP（全对最短路径）在交通规划、社交网络分析等众多领域中的基础性地位。
然而，其固有的高算法复杂度——经典Floyd-Warshall算法的时间复杂度为O(n³)，空间复杂度为O(n²)——使其在处理大规模图时变得异常困难，
主要的瓶颈在于海量数据移动远超传统存储层次的带宽能力。
接着，文章分析了现有解决方案的不足：基于GPU的并行计算虽然能提供高吞吐量，但需要庞大的硬件资源和密集的节点间通信，导致高昂的能耗和通信开销；
而早期的内存处理（PIM）架构大多为稀疏图遍历（如BFS）优化，其原子操作和轻量级数据通路无法有效应对APSP所需处理的密集距离矩阵和复杂的最小-加法（Min-Plus）矩阵乘法。
基于这些挑战，本文提出了RAPID-Graph，一个集成了算法、架构和器件层面创新的软硬件协同设计PIM系统，旨在从根本上解决精确APSP计算的瓶颈。

**第二章：背景 (Background)**
本章为后续内容铺垫了基础知识。首先，介绍了图的两种数据结构：邻接矩阵和压缩稀疏行（CSR）。
对于存储而言，CSR格式更节省空间，因为它只记录存在的边。但对于计算，尤其像FW算法这类需要频繁访问任意顶点对距离的场景，
使用密集的邻接矩阵在数据访问上更为规整和高效。
随后，详细回顾了经典的FW动态规划算法，其核心思想是通过迭代地引入中间顶点`k`来松弛所有顶点对`(i, j)`之间的路径，
更新公式为 `D[i][j] = min(D[i][j], D[i][k] + D[k][j])`。
为了将此算法扩展到无法在单个设备上处理的大规模图，文章介绍了分块APSP（Partitioned APSP）的四步法，
这是一种“分而治之”的策略，包括：1. 组件内APSP计算；2. 提取边界顶点构成边界图并计算其APSP；
3. 将边界图结果注入回各个组件；4. 通过Min-Plus合并计算最终的跨组件路径。这个方法是本文所提递归算法的基石。
最后，简要介绍了相变存储器（PCM）技术，它利用材料在晶态和非晶态之间的电阻差异来存储数据，具有非易失性、高密度和快速读访问的特点，
非常适合作为执行就地计算（In-place Computing）的PIM技术载体。

**第三章：RAPID-Graph软硬件协同设计 (RAPID-Graph Software-Hardware Co-design)**
本章是论文技术贡献的核心，详细阐述了RAPID-Graph的软硬件协同设计。
*   **A. 递归分块APSP算法**: 这是本文在算法层面的关键创新。它将分块APSP的思想推广为递归形式。
    当图的规模超过单个PIM瓦片（Tile）的处理上限（本文设定为1024个顶点）时，算法会使用图划分工具（如METIS）将其划分为若干组件。
    所有组件的边界顶点被提取出来，形成一个边界图。如果这个边界图的规模仍然超过上限，那么就对这个边界图本身进行再次（递归地）划分。
    这个过程持续进行，直到最高层级的边界图也可以被单个PIM瓦片处理。计算过程则自底向上，从最细粒度的划分开始，
    逐层计算局部APSP和边界图APSP，并将结果向上传播。这种递归结构极大地提升了并行度，因为它允许大量PIM瓦片在没有全局同步的情况下独立工作。

*   **B. 异构体系结构**: 描述了一个为递归算法量身定制的2.5D多芯粒（Multi-Die）硬件系统。该系统由以下部分构成：
    1.  **PCM-FW Die**: 一个专门执行FW算法的PIM裸片，负责计算组件内部和各级边界图内部的APSP。
    2.  **PCM-MP Die**: 一个专门执行Min-Plus合并操作的PIM裸片，用于计算跨组件的最终最短路径。
    3.  **Logic Base Die**: 作为中央控制器，负责管理整个系统的数据流，包括从外部存储读取数据、在CSR和密集矩阵之间进行格式转换，以及调度任务到不同的PIM裸片。
    4.  **HBM3**: 高带宽内存，用作高速缓存和暂存区，存放当前正在处理的子图、中间结果等热数据。
    5.  **UCIe (Universal Chiplet Interconnect Express)**: 一种高速芯粒互联技术，以2 Tb/s的带宽连接上述所有芯粒，确保低延迟和高吞吐量的数据交换。
    6.  **FeNAND**: 位于PCB板上的大容量（16TB）外部存储，用于持久化存储原始图数据和最终计算出的全局APSP结果。

*   **C. 硬件设计与实现**: 深入探讨了PIM裸片内部的微架构。FW和MP裸片都由130个并行的PIM单元（Macro）组成，
    每个单元基于1024x1024的1T1R PCM阵列。
    *   **PCM-FW瓦片**的核心是一个**置换单元（Permutation Unit）**。在FW算法的每次迭代中，都需要选定一个主元（pivot），并用其对应的行和列去更新矩阵的其余部分。
        该置换单元通过专门的硬件逻辑（如行缓冲、重排序缓冲和DMA引擎），可以高效地在PIM阵列内部完成数据的重排和提取，
        使得主块（Main Block）的更新可以大规模并行执行，同时通过流水线操作隐藏了数据移动的延迟。
    *   **PCM-MP瓦片**的核心是一个**流水线化的最小比较器树（Min-Comparator Tree）**。在MP合并步骤中，需要为每个顶点对计算`min(D[i,k] + D[k,j])`，
        这本质上是一个多路最小值问题。该硬件单元采用6级树状结构，结合32个并行的进位超前加法器（CLA），
        能在短短13个时钟周期内从1024个候选路径中找出最短的一条，极大地加速了MP合并过程。

*   **D. 映射与调度方案**: 描述了如何将算法高效地映射到硬件上。FW更新被巧妙地分解和重排，
    通过逻辑上分离主元行、主元列和主更新块，使得PIM阵列的利用率最大化。MP合并操作则被设计为一个两阶段的流水线过程，
    在PCM-MP瓦片上高效执行。整个数据流被精心调度，从FeNAND到HBM3，再到各个PIM裸片，最后写回，形成一个高效的数据处理通路。

**第四章：实验结果与分析 (Experimental Results and Analysis)**
本章通过一个自研的、考虑了器件级别细节的周期精确模拟器，对RAPID-Graph的性能进行了全面评估。
*   **性能和能效对比**: 在包含高达32768个节点的合成图上，与一颗高端Intel CPU（i7-11700K）相比，RAPID-Graph实现了最高1061倍的加速和7208倍的能效提升。
    与顶级的NVIDIA H100 GPU相比，也取得了高达42.8倍的加速和392倍的能效提升。这一巨大优势源于RAPID-Graph通过PIM技术从根本上消除了由O(n³)数据移动所引发的内存带宽瓶颈。
*   **与SOTA方案对比**: 在包含245万个节点的真实世界图（OGBN-Products）上，RAPID-Graph的性能也远超现有的、基于大型GPU集群的分布式APSP解决方案
    （如Partitioned APSP和Co-Parallel APSP）。例如，它比Co-Parallel APSP快5.8倍，比Partitioned APSP节能1186倍。
    其核心优势在于避免了分布式系统中昂贵的节点间通信和同步开销。
*   **可扩展性分析**:
    *   **随规模扩展**: RAPID-Graph的执行时间与图的节点数成线性关系，展现了优异的可扩展性。相比之下，H100 GPU在节点数超过1024后，
        由于缓存不堪重负，其性能和能效急剧恶化。
    *   **随度数扩展**: 结果显示，当内存充足时，图的边数（即节点度）对精确APSP的计算时间影响不大，RAPID-Graph和H100都表现稳定。
    *   **对拓扑的适应性**: RAPID-Graph在处理具有社群结构的图（如NWS图和真实世界的OGBN图）时，比处理随机图（ER图）更高效。
        这是因为结构化的图在划分后能产生更小、更稀疏的边界，从而显著减少了计算量最大的边界图APSP步骤的工作负载。这体现了其分区算法的拓扑感知优势。

**第五章：结论 (Conclusion)**
本章总结了RAPID-Graph作为首个专为精确APSP问题设计的软硬件协同PIM系统的核心贡献和成就。
文章重申了系统的两大创新支柱：1）**递归分块算法**，它将大规模问题分解为适合PIM阵列处理的小规模任务，并最大化了并行性；
2）**异构2.5D硬件架构**，它通过专门的PIM裸片、高带宽内存和高速互联技术，为算法提供了强大的硬件支持。
最终，RAPID-Graph以远低于大型数据中心集群的成本、功耗和运行时间，实现了前所未有的APSP计算性能。
它不仅在各项指标上全面超越了顶尖的CPU和GPU，还在大规模真实世界图上击败了现有的分布式计算方案，
为解决大规模图分析中的核心瓶颈问题提供了一条极具前景的技术路径。
