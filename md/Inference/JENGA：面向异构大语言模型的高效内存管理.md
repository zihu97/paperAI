# JENGA：面向异构大语言模型的高效内存管理

## 中文摘要

大型语言模型（LLM）被广泛使用，但运行成本高昂，尤其是在推理工作负载不断增长的情况下。为了降低成本，通过有效管理 GPU 内存来最大化请求批处理大小至关重要。
虽然最近提出的 PagedAttention 旨在提高内存管理的效率，但我们发现，现代 LLM 架构在嵌入维度、注意力和访问模式上日益增长的异构性给内存分配带来了新的挑战。
在本文中，我们提出了 JENGA，一个用于 LLM 中异构嵌入的新型内存分配框架。JENGA 解决了两个关键挑战：
（1）在管理不同大小的嵌入时最小化内存碎片，以及（2）支持为各种层的特定令牌依赖模式量身定制的灵活缓存和驱逐策略。
JENGA 采用了一个两级内存分配器，利用嵌入大小的最小公倍数（LCM）来优化内存使用，并提供 API 来表达特定于层的缓存逻辑以增强内存重用。
我们在 vLLM（一个最先进的 LLM 推理引擎）上实现了 JENGA，并用多样化的 LLM、数据集和 GPU 配置对其进行了评估。
评估结果表明，JENGA 将 GPU 内存利用率提高了高达 79.6%，并将服务吞吐量提高了高达 4.92 倍（平均 1.80 倍）。

## 各章节主旨内容

*   **第一章：引言**
    *   介绍了 LLM 服务中 GPU 内存管理的重要性，并指出现有的 PagedAttention 机制在处理日益异构的现代 LLM 架构时面临的挑战。
    *   这些异构性体现在嵌入维度、注意力机制和内存访问模式上。论文提出了 JENGA 框架来应对这些挑战。

*   **第二章：背景**
    *   解释了 LLM 推理中的基本概念，特别是 KV 缓存机制。LLM 是自回归模型，需要存储之前所有令牌的键（Key）和值（Value）对，即 KV 缓存，这占用了大量的 GPU 内存。

*   **第三章：异构 LLM 及其挑战**
    *   详细阐述了现代 LLM 的四种异构性来源：稀疏注意力（如滑动窗口注意力）、状态空间模型（如 Mamba）、多模态输入（如视觉嵌入）以及服务多并发模型（如思辨解码）。
    *   分析了这些异构性如何导致内存碎片和前缀缓存（Prefix Caching）的复杂性。

*   **第四章：两级内存分配**
    *   介绍了 JENGA 的核心设计：一个两级内存分配器。
    *   底层使用固定大小的“大页”，其大小是所有不同嵌入大小的最小公倍数（LCM）。
    *   顶层则根据每个层（Layer）的具体需求，将“大页”划分为不同大小的“小页”。
    *   同时，引入了“请求感知”的分配策略，以减少内部碎片。

*   **第五章：可定制的前缀缓存**
    *   描述了 JENGA 如何通过提供统一的 API 来支持不同层类型的定制化缓存驱逐（Eviction）和命中（Hit）规则。
    *   通过平衡（Balanced）和对齐（Aligned）的驱逐策略，确保在不同层之间高效、一致地管理缓存。

*   **第六章：案例研究**
    *   展示了 JENGA 如何自然地扩展以支持思辨解码（Speculative Decoding）和多模型服务。
    *   分析了 JENGA 在处理视觉语言模型（VLM）的视觉嵌入缓存时的优势，能够有效减少峰值内存消耗。

*   **第七章：评估**
    *   在 NVIDIA H100 和 L4 GPU 上，将 JENGA 与基线 vLLM 进行了端到端性能比较。
    *   实验结果表明，JENGA 在多种异构 LLM（如 Llama 3.2 Vision, Gemma-2, Jamba 等）上显著提升了吞吐量（平均 1.8 倍，最高 4.92 倍）和内存利用率（最高 79.6%），而没有牺牲延迟。

*   **第八章：相关工作**
    *   回顾了 LLM 服务系统和 LLM 内存分配与优化的相关研究，强调了 JENGA 作为一个通用解决方案，与许多特定于模型类型的优化方法不同。

*   **第九章：结论**
    *   总结了 JENGA 作为一个高效的内存分配框架，通过两级分配器和可定制的缓存策略，成功解决了现代 LLM 架构中的异构性挑战，显著提升了服务性能。

## 指定章节详解

### 第一章：引言

本章首先强调了在 LLM 服务中，GPU 内存是限制系统吞吐量的关键瓶颈。为了提高 GPU 利用率，业界普遍采用批处理（Batching）技术，但这需要高效的内存管理来存储大量请求的 KV 缓存。
接着，指出了由 OpenAI 发明的 PagedAttention 技术虽然极大地改善了内存碎片问题，但其设计基于两个基本假设：1）所有嵌入（Embeddings）大小固定；2）生成下一个令牌需要依赖于所有前面的令牌（Full-prefix dependency）。
然而，现代 LLM 架构（如混合注意力模型、多模态模型）已经打破了这两个假设，导致 PagedAttention 效率下降。
因此，本文提出了 JENGA，一个新的内存管理框架，旨在解决这些新出现的异构性问题。JENGA 的核心贡献在于其两级内存分配器和灵活的缓存策略，能够有效处理不同大小的嵌入，并为不同层定制缓存逻辑。

### 第二章：背景

本章为不熟悉 LLM 推理的读者提供了基础知识。核心概念是 **KV 缓存**。在自回归生成过程中，模型每生成一个新令牌，都需要计算当前令牌与所有先前令牌之间的注意力。为了避免重复计算，系统会将先前每个令牌在每个注意力层（Attention Layer）产生的键（Key）和值（Value）张量存储在 GPU 内存中，这就是 KV 缓存。随着序列长度和批处理大小的增加，KV 缓存会占用数十 GB 的内存，成为推理过程中的主要内存开销。因此，高效管理 KV 缓存是提升 LLM 服务性能的关键。

### 第三章：异构 LLM 及其挑战

本章深入分析了现代 LLM 架构中日益增长的异构性，这些异构性给内存管理带来了新的挑战。

1.  **异构的注意力机制**：
    *   **稀疏注意力（Sparse Attention）**：如滑动窗口注意力（Sliding-window Attention），只关注最近的一部分令牌，而不是全部前缀。Gemma-2 等模型混合使用全注意力（Full Attention）和滑动窗口注意力。
    *   **状态空间模型（State Space Models, SSM）**：如 Mamba，使用一个固定大小的状态张量来代替完整的 KV 缓存，这个状态张量的大小通常远大于单个令牌的 KV 缓存。Jamba 等模型则混合了 Mamba 层和传统的自注意力层。

2.  **异构的数据模态**：
    *   **多模态语言模型（VLM）**：如 LLaVA，除了处理文本令牌，还需要处理来自视觉编码器的图像嵌入。这些图像嵌入和文本令牌的 KV 缓存大小通常是不同的。

3.  **异构的模型服务**：
    *   **思辨解码（Speculative Decoding）**：同时使用一个小的“草稿”模型和一个大的“验证”模型。这两个模型的 KV 缓存大小不同，需要同时管理。

这些异构性导致了两个主要挑战：
*   **内存碎片（Memory Fragmentation）**：当需要为不同大小的嵌入分配内存时，固定大小的页面会造成严重的内存浪费（内部碎片）。论文以 Llama 3.2 Vision 模型为例，指出内存浪费可高达 79.6%。
*   **复杂的前缀缓存（Prefix Caching）**：不同层对前缀的依赖不同（例如，滑动窗口层只依赖窗口内的令牌），这要求缓存驱逐策略也必须是可定制的，否则会导致缓存命中率下降。同时，不同层之间的驱逐策略需要保持平衡和对齐，以最大化整体的缓存命中率。

### 第四章：两级内存分配

为了解决内存碎片问题，JENGA 设计了一个创新的两级内存分配系统。

*   **第一级：LCM 分配器（LCM Allocator）**
    *   这一层将整个 GPU KV 缓存区划分为一系列连续的、固定大小的“大页”（Large Pages）。
    *   这个“大页”的大小被设定为模型中所有不同类型嵌入（如自注意力 KV 缓存、Mamba 状态、视觉嵌入等）大小的**最小公倍数（Least Common Multiple, LCM）**。
    *   例如，如果一个模型有两种嵌入，大小分别为 2KB 和 3KB，那么 LCM 分配器管理的“大页”大小就是 6KB。
    *   这种设计确保了每个“大页”都可以被无缝地、无浪费地划分为任何一种类型的嵌入所需的“小页”。

*   **第二级：定制化分配器（Customized Allocators）**
    *   对于每种类型的层（如自注意力层、Mamba 层），都有一个对应的定制化分配器。
    *   当需要为特定层分配内存时，该层的分配器会向 LCM 分配器申请一个或多个“大页”，然后将其内部划分为适合该层嵌入大小的“小页”（Small Pages）。
    *   所有的内存分配请求都发生在“小页”层面。

*   **请求感知分配（Request-Aware Allocation）**
    *   为了进一步减少内部碎片，JENGA 尝试将同一个请求（Request）的“小页”分配在同一个“大页”内。
    *   这样做的好处是，当一个请求完成并释放其所有“小页”时，它所在的“大页”很可能被完全清空，从而可以被 LCM 分配器回收并用于其他请求，避免了不同请求的内存块交错导致的碎片问题。

通过这种两级结构，JENGA 既能像 PagedAttention 一样通过分页来管理内存，又能灵活地处理不同大小的嵌入，从而最大限度地减少内存浪费。

### 第五章：可定制的前缀缓存

为了应对异构依赖性带来的前缀缓存挑战，JENGA 提供了一套灵活的机制。

*   **统一的缓存管理接口**：
    *   JENGA 定义了一个名为 `LayerSupportsPrefixCache` 的接口，其中包含 `update_last_access`、`set_prefix_length` 和 `get_possible_prefix` 等方法。
    *   不同的层类型（如滑动窗口层、Mamba 层）可以实现这个接口，以定义自己独特的缓存驱逐和命中逻辑。

*   **定制化缓存驱逐（Customizable Cache Eviction）**：
    *   **平衡驱逐**：通过 `update_last_access` 接口，JENGA 可以根据每种层的特性来更新令牌的“最后访问时间”。例如，在滑动窗口层中，只有窗口内的令牌才被认为是“最近访问”的。这确保了系统会优先驱逐那些在所有层中都变得不重要的令牌，从而实现了跨层类型的平衡驱逐。
    *   **对齐驱逐**：通过 `set_prefix_length` 接口，可以为具有相同访问时间戳的页面分配不同的驱逐优先级，确保不同层驱逐的是逻辑上相同的令牌集，避免了驱逐策略的错位。

*   **定制化缓存命中（Customizable Cache Hit）**：
    *   当一个新请求到达时，JENGA 会调用每个层类型的 `get_possible_prefix` 方法，来确定在该层的缓存策略下，哪些前缀是有效的。
    *   然后，系统会选择在**所有层类型中都有效的最长公共前缀**作为最终的缓存命中结果。

通过这套机制，JENGA 能够智能地管理异构模型的前缀缓存，最大化缓存命中率，从而减少不必要的重计算，提升系统吞吐量。

### 第六章：案例研究

本章探讨了 JENGA 在两个具体高级应用场景中的实现和优势。

1.  **思辨解码（Speculative Decoding）和多模型服务**：
    *   思辨解码需要同时管理一个小型“草稿”模型和一个大型“验证”模型的 KV 缓存，这两种缓存的大小通常不同。
    *   JENGA 的两级分配机制可以自动处理这两种不同大小的 KV 缓存，无需为这种情况进行专门的硬编码设计，从而实现了低碎片的内存管理。
    *   这个能力可以自然地推广到在单个 GPU 上服务多个不同模型的场景。

2.  **视觉语言模型（VLM）的视觉嵌入缓存**：
    *   VLM 的推理过程通常分为两步：首先，视觉编码器将输入图像转换为一系列视觉嵌入；然后，语言模型将这些嵌入与文本一起处理。
    *   传统的推理引擎（如 vLLM）在处理长序列时，可能会因为内存不足而分块（Chunked）处理输入。在这种情况下，它们可能会多次运行视觉编码器，或者在每次分块时都为视觉嵌入分配 KV 缓存，导致内存峰值增加。
    *   JENGA 能够将视觉嵌入缓存视为一种独立的内存类型进行管理。它可以在第一次计算后就缓存视觉嵌入，并在后续的语言模型处理步骤中重用它们，而无需重新计算或占用额外的 KV 缓存空间。这显著降低了峰值内存使用，并完全消除了视觉编码器的重复计算开销。

### 第七章：评估

本章通过详尽的实验来验证 JENGA 的性能。

*   **实验设置**：
    *   **平台**：NVIDIA H100 (80GB) 和 L4 (24GB) GPU。
    *   **基线**：vLLM v0.6.3，一个业界领先的 LLM 推理引擎。JENGA 是在 vLLM 基础上实现的，仅替换了其内存管理部分。
    *   **模型**：涵盖了多种异构架构，包括 Llama 3.2 Vision (多模态), Gemma-2 (滑动窗口), Jamba-1.5 (Mamba混合), Character.ai 模型 (私有稀疏注意力) 等。
    *   **数据集**：MMMU-pro (多模态), arXiv-QA (长文本问答)。

*   **主要发现**：
    *   **端到端吞吐量**：在 H100 上，JENGA 相较于 vLLM 实现了平均 1.80 倍，最高 4.92 倍的吞吐量提升。在 L4 GPU 上也有类似的显著提升。这种提升主要来自于更低的内存浪费和更高效的前缀缓存。
    *   **内存碎片分析**：在 Ministral 模型上，vLLM 由于无法有效处理滑动窗口注意力，导致了 38.2% 的内存浪费。而 JENGA 通过其灵活的分配策略，将内存浪费减少到几乎为零（0.04%）。
    *   **前缀缓存效率**：在处理多篇文章的问答任务时，当文章数量增加，vLLM 的缓存系统迅速饱和，而 JENGA 的定制化驱逐策略使其缓存命中率比 vLLM 高出 1.60 倍，从而带来了 1.77 倍的吞吐量提升。
    *   **案例研究性能**：在 VLM 任务中，JENGA 对视觉嵌入的有效缓存带来了 1.88 倍的吞吐量提升。在思辨解码任务中，JENGA 在异构模型上的性能比 vLLM 的手动优化策略平均高出 1.58 倍。

*   **开销分析**：在标准的同构模型（如 Llama 3.1）上，JENGA 的性能与 vLLM 相当，证明其引入的额外开销可以忽略不计。

### 第八章：相关工作

本章将 JENGA 与现有的 LLM 服务优化工作进行了比较。
*   **LLM 服务系统**：许多工作如 Orca, DeepSpeed-FastGen, SGLang 等都致力于优化 LLM 服务，但它们主要关注同构模型。JENGA 可以作为这些系统的底层内存管理器，将它们的优化能力扩展到异构模型。
*   **LLM 内存分配**：PagedAttention 是里程碑式的工作，但未能处理异构性。vAttention 使用 GPU 虚拟内存，但开销较大且无法实现灵活的前缀缓存。其他工作如 SLORA, Marconi, SmartSpec 都是为特定模型或场景（如 LoRA, Mamba, 思辨解码）设计的专用解决方案。
*   **LLM 内存优化**：FlashAttention 等技术通过算子融合减少了内存占用。一些工作研究了 KV 缓存的卸载（Offloading）和剪枝（Pruning）。JENGA 与这些技术是互补的，可以集成它们来进一步提升性能。

相比之下，JENGA 提供了一个**通用且可扩展**的解决方案，能够适应广泛的现代 LLM 架构，而不仅仅是针对某个特定的模型或优化点。

### 第九章：结论

论文最后总结道，JENGA 是一个为现代异构 LLM 设计的高效内存分配框架。通过其创新的两级内存分配器（利用 LCM 优化）和可定制的缓存策略，JENGA 成功地解决了由不同嵌入大小和复杂依赖关系引起的内存碎片和缓存管理难题。实验证明，JENGA 在不牺牲延迟的情况下，显著提高了 GPU 内存利用率和服务吞吐量，为未来日益多样化的 LLM 架构的高效服务提供了一个强大而通用的解决方案。
