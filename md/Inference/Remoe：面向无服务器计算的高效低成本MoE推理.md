# Remoe：面向无服务器计算的高效低成本MoE推理

## 第一部分：论文整体骨架

### 1. 论文基本信息
*   **英文标题**：Remoe: Towards Efficient and Low-Cost MoE Inference in Serverless Computing
*   **中文标题**：Remoe：面向无服务器计算的高效低成本MoE推理
*   **作者**：Wentao Liu, Yuhao Hu, Ruiting Zhou, Baochun Li, Ne Wang
*   **机构**：东南大学计算机科学与工程学院，多伦多大学电气与计算机工程系，香港理工大学计算学系
*   **文件名**：2512.18674v1.pdf

### 2. 中文摘要
混合专家模型（MoE）因其能够通过稀疏专家激活来扩展模型容量，已成为大型语言模型（LLM）的主流架构。与此同时，无服务器计算凭借其弹性伸缩和按量付费的特性，非常适合部署具有突发负载的MoE模型。然而，MoE模型中大量的专家会导致因内存密集的参数缓存而产生高昂的推理成本。由于输入依赖的专家激活特性，简单的模型分区难以缓解这些成本。

为了解决这些问题，本文提出了 **Remoe**，一种专为无服务器计算量身定制的异构MoE推理系统。Remoe将非专家模块分配给GPU，将专家模块分配给CPU，并进一步将不频繁激活的专家卸载到独立的无服务器函数中，以减少内存开销并实现并行执行。我们整合了三项关键技术：（1）**相似提示搜索（SPS）算法**，基于输入的语义相似性预测专家激活模式；（2）**主模型预分配（MMP）算法**，通过最坏情况内存估计确保服务水平目标（SLO）；（3）**联合内存与副本优化框架**，利用拉格朗日对偶性和最长处理时间（LPT）算法进行优化。我们在Kubernetes上实现了Remoe，并在多个LLM基准测试中对其进行了评估。实验结果表明，与最先进的基准相比，Remoe降低了高达 **57%** 的推理成本，并缩短了 **47%** 的冷启动延迟。

### 3. 各章节主旨内容

*   **I. 绪论 (Introduction)**
    *   背景：LLM和MoE的兴起，以及无服务器计算作为推理部署范式的流行。
    *   挑战：无服务器定价模型下，MoE全量加载的内存浪费和高昂成本；现有卸载方法在无服务器环境下的局限性；专家激活的不平衡性和不可预测性。
    *   解决方案：提出Remoe，一种异构推理架构。核心思想是利用GPU处理非专家模块，CPU处理专家模块，并根据激活频率将低频专家拆分到远程无服务器函数。
    *   贡献：设计了异构架构、SPS预测算法、MMP资源预分配算法、成本-延迟优化框架，并进行了原型实现和评估。

*   **II. 动机 (Motivation)**
    *   **部分专家激活**：在无服务器计费模式下，未激活的专家占用内存会造成巨大浪费。现有的专家卸载方法（CPU-GPU互换）仍需在CPU上维护大内存池，不适合无服务器环境。
    *   **层间通信开销**：AWS Lambda等平台的Payload限制（如6MB）远大于LLM层间传输的Token数据量（KB级别），使得将低频专家卸载到独立函数而不经过中间存储成为可能。
    *   **CPU上的专家推理**：非专家模块（Attention等）计算密集适合GPU，而专家模块（FFN）计算量较小且稀疏，适合在便宜的CPU上执行，从而降低成本。

*   **III. 系统模型 (System Model)**
    *   **系统概览**：定义了MoE推理的四个阶段（预处理、预填充、解码、后处理）。Remoe将所有中间层打包为主模型函数（非专家在GPU，部分专家在CPU），低频专家作为“远程专家”部署在独立的CPU无服务器函数中。
    *   **推理延迟模型**：详细推导了预填充（Prefilling）和解码（Decoding）阶段的延迟，包括本地计算、数据传输和远程计算时间。
    *   **推理成本模型**：成本由主模型成本和远程专家成本组成，主要取决于内存规格和运行时间（符合无服务器定价）。
    *   **问题形式化**：建立了一个在满足SLO（TTFT和TPOT）约束下最小化总推理成本的优化问题。

*   **IV. Remoe设计 (Remoe Design)**
    *   **核心理念**：通过请求到达时的激活预测、资源预分配、远程专家选择、内存优化和多副本推理五个步骤来处理请求。
    *   **激活分布预测**：提出SPS算法，利用聚类树和软余弦相似度（SCS）快速检索历史提示，并加权预测新请求的专家激活分布。
    *   **主模型资源预分配**：提出MMP算法，基于概率界限理论（定理1）计算最坏情况下的处理时间，从而确定满足SLO的主模型最小内存规格。
    *   **远程专家选择**：基于效用分数（Utility Score）决定哪些专家保留在本地，哪些卸载到远程。
    *   **远程专家内存优化**：通过松弛整数变量和凸性分析，利用拉格朗日对偶法求解最优内存配置。

*   **V. 评估 (Evaluation)**
    *   **设置**：在Kubernetes上搭建原型，使用A100 GPU和Xeon CPU。数据集包括LMSYS-Chat-1M等，模型使用GPT2-moe和Deepseek-v2-lite。
    *   **预测准确性**：SPS算法在语义相似度搜索上比现有方法（如VarPAM, BF）更快且更准确。
    *   **整体性能**：相比纯CPU、纯GPU和Fetch（理想化卸载）等基准，Remoe在不同模型和负载下均实现了最低成本。
    *   **冷启动**：Remoe通过并行启动和重叠策略，显著降低了冷启动时间（减少约57%）。

*   **VI. 相关工作 (Related Work)**
    *   回顾了无服务器LLM推理（关注冷启动、资源分配）和GPU显存受限MoE推理（关注卸载、量化、预测）的现有研究。指出Remoe是首个在无服务器GPU-CPU混合架构下系统解决MoE推理成本效益的工作。

*   **VII. 结语 (Concluding Remarks)**
    *   总结了Remoe的异构系统设计及其算法创新。强调了其在降低成本和冷启动延迟方面的显著效果。未来工作将关注容错性和网络波动处理。

### 4. 中文结论
为了最小化推理成本，本文提出了异构系统 **Remoe**。我们设计了用于专家激活预测、资源预分配以及联合内存-副本优化的算法。我们在Kubernetes上的Remoe实现表明，它显著降低了推理成本和冷启动延迟。目前的方法依赖于对无服务器环境的理想化假设。因此，我们未来的工作将侧重于设计一个高度容错的系统，以应对现实世界的运维复杂性，如不可预测的冷启动时间和网络延迟波动。

## 第二部分：指定章节详解

### I. 绪论 (Introduction) & II. 动机 (Motivation)
**设计思想与背景：**
随着LLM向MoE架构演进，模型参数量剧增但激活参数量相对较小。无服务器计算（Serverless）按量付费的模式理论上适合MoE的突发访问特性。然而，MoE庞大的参数量导致在Serverless中即使不激活也需要为加载所有专家付费（内存占用大）。
**现有问题：**
1.  **内存浪费**：在GPU上部署全量MoE非常昂贵，且大量专家闲置。
2.  **现有卸载方案不适用**：传统的CPU-GPU卸载（如fMoE）需要维护一个常驻的CPU内存池，不符合Serverless“用完即走”的理念。
3.  **预测困难**：专家激活高度依赖输入，且动态变化，难以提前准确规划资源。

**核心洞察（Motivation）：**
1.  **异构计算的经济性**：专家层（FFN）计算量小，适合CPU；非专家层（Attention）计算密集，适合GPU。CPU比GPU便宜得多。
2.  **Payload“红利”**：AWS Lambda等平台的函数间调用Payload限制（如6MB）远大于LLM层间传递的Token Embedding大小（仅KB级别）。这意味着可以直接将Token发给远程CPU函数处理，而不需要慢速的外部存储（S3）。
3.  **拆分的可能性**：由于上述两点，可以将低频专家拆分到独立的、按需启动的CPU Serverless函数中。

### III. 系统模型 (System Model)
**架构设计：**
Remoe将推理分为四个阶段：
1.  **预处理**：Tokenization。
2.  **预填充（Prefilling）**：处理Prompt的所有Token。
3.  **解码（Decoding）**：逐个生成Token。
4.  **后处理**：Detokenization。

**组件划分：**
*   **主模型（Main Model）**：包含所有非专家模块（Attention, Gate等）和高频“本地专家”。部署在配备GPU的Serverless实例上（GPU处理非专家，CPU处理本地专家）。
*   **远程专家（Remote Experts）**：低频专家，部署在纯CPU的Serverless实例上。
*   **交互方式**：主模型在遇到需要远程专家的Token时，直接通过网络调用远程专家函数。

**数学建模：**
论文详细建立了延迟（Latency）和成本（Cost）模型。
*   **延迟**：包含GPU计算、CPU计算（本地/远程）、数据传输时间。特别考虑了Serverless的冷启动和网络带宽限制。
*   **成本**：$Cost = Memory 	imes Time$。这是Serverless的标准计费公式。优化目标就是在满足延迟约束（TTFT, TPOT）的前提下，最小化 $\sum (Memory \times Time)$。

### IV. Remoe设计 (Remoe Design) - 核心技术详解

#### 1. 相似提示搜索 (SPS) 与激活预测
**挑战**：如何快速准确地预测新Prompt会激活哪些专家？
**SPS算法**：
*   **聚类树构建**：离线阶段，基于历史Prompt的语义嵌入（Embedding）构建多叉聚类树。使用软余弦相似度（SCS）作为距离度量，因为它能捕捉文本的语义重叠。
*   **在线搜索**：新请求到达时，在树中搜索最相似的叶子节点，获取Top-$\alpha$个相似的历史Prompt。
*   **加权预测**：利用这$\alpha$个历史Prompt的专家激活分布，加权求和得到新Prompt的预测激活分布矩阵 $\tilde{S}$。
*   **优势**：相比暴力搜索（Brute Force），速度快几个数量级（毫秒级）；相比纯基于分布的方法，语义相似性更能反映真实的激活模式。

#### 2. 主模型资源预分配 (MMP)
**挑战**：Serverless需要预先申请资源（内存大小），且冷启动需要时间。如何在预测完成前就开始启动主模型，且保证资源够用？
**MMP算法**：
*   **并行启动**：在进行SPS预测的同时，根据理论推导的最坏情况启动主模型。
*   **理论界限**：利用霍夫丁不等式（Hoeffding’s inequality）推导出一个定理（Theorem 1）：当$n$个Token通过$m$个专家时，单个专家处理的Token数量在95%概率下不会超过某个界限。
*   **资源计算**：基于这个界限，计算满足TTFT（首字延迟）和TPOT（吞吐量）约束所需的最小主模型内存 $w_v$。这保证了即使流量分布极端，系统也不会崩溃或超时。

#### 3. 远程专家选择与内存优化
**决策变量**：
*   $b$：远程专家比例（多少专家放远程）。
*   $y_{l,v}$：远程专家的内存规格。
*   $z_l$：远程专家的副本数量。

**优化流程**：
1.  **选择远程专家**：计算每个专家的“效用分数”（Utility Score），即在预填充和解码阶段的预期计算量。效用最低的专家被标记为远程专家。
2.  **凸优化问题转化**：原始的内存选择问题是一个非线性整数规划问题（NP-hard）。作者将其松弛为连续变量问题，并证明了其在特定区间内的凸性（Convexity）。
3.  **拉格朗日对偶法**：利用凸性，通过拉格朗日乘数法快速求解最优的内存配置 $y^*$。这比启发式搜索更精确且高效。

#### 4. 多副本推理 (Multi-replicas)
**场景**：远程专家可能包含多个专家模块，且Token路由是动态的。如果所有远程专家都在一个实例里，可能会处理不过来；如果分太多实例，冷启动和通信开销会增加。
**解决方案**：
*   **LPT算法**：将远程专家集合 $\mathcal{R}_l$ 划分为 $z_l$ 个子集（副本）。这是一个多路划分问题（Multiway Number Partitioning）。Remoe使用最长处理时间（LPT）算法，优先分配负载最重的专家，以实现负载均衡。
*   **副本数量决策**：通过迭代增加副本数量 $z_l$，直到满足最坏情况下的TPOT约束。

### V. 评估 (Evaluation)
**实验亮点：**
1.  **对比基准**：
    *   **CPU/GPU**：纯CPU或纯GPU部署。
    *   **Fetch**：理想化的按需加载（假设无延迟），作为理论下界。
    *   **MIX**：混合部署但无动态卸载。
2.  **成本降低**：在Deepseek-v2-lite（大型MoE）上，Remoe相比GPU基准降低了 **57.14%** 的成本。这主要归功于将大量闲置专家移到了廉价的CPU Serverless函数中，且按需计费。
3.  **冷启动优化**：Remoe的冷启动时间比标准部署低 **47%**。原因在于它将大模型拆分了，主模型变小了，启动更快；同时远程专家可以并行启动。
4.  **预测性能**：SPS算法在JS散度（衡量分布差异）上优于现有SOTA方法（如VarPAM），且构建树和搜索的时间极短。

### VI. 总结与展望
Remoe通过精细的软硬件协同设计，成功解决了MoE在Serverless上的“水土不服”问题。它不仅利用了Serverless的价格优势，还通过算法克服了通信和延迟的劣势。这是一个典型的系统级优化工作，通过拆分、预测、调度和数学规划，实现了AI推理的降本增效。未来的方向是增强系统的鲁棒性，以应对公有云环境的不确定性。

### 4. 中文结论
为了最小化推理成本，本文提出了异构系统 **Remoe**。我们设计了用于专家激活预测、资源预分配以及联合内存-副本优化的算法。我们在Kubernetes上的Remoe实现表明，它显著降低了推理成本和冷启动延迟。目前的方法依赖于对无服务器环境的理想化假设。因此，我们未来的工作将侧重于设计一个高度容错的系统，以应对现实世界的运维复杂性，如不可预测的冷启动时间和网络延迟波动。
